# 数据集分析报告撰写 Skill

本skill提供撰写数据集分析报告的完整工作流程。

## 工作流程

### Step 1: 探索数据集结构

1. 查找目录中的 README 文件（README.md, README_zh.md）
2. 定位数据文件（.jsonl, .csv, .json等）
3. 查找论文PDF文件
4. 分析代码文件结构

### Step 2: 分析数据内容

使用分析脚本自动分析 JSONL 数据集：

```bash
python scripts/analyze_dataset.py /path/to/dataset.jsonl [-o output_report.md] [-n top_n_categories]
```

脚本会自动：
- 读取数据文件，解析JSON结构
- 统计字段分布、类别数量、数据量
- 分析数据长度特征
- 生成 Markdown 分析报告

### Step 3: 提取论文关键信息

从 README 和论文中提取：
- **来源**：发布机构、时间、平台
- **目标**：解决什么问题
- **应用场景**
- **数据量与分类**
- **评测方法**（评测指标、提示词模板）

### Step 4: 分析评测代码

1. 查看评测脚本中的提示词模板
2. 分析评判逻辑和评分方法

### Step 5: 撰写报告

按照模板结构撰写报告：
- 1. 简介（来源、目标、应用场景、数据集描述）
- 2. 数据集能力体系（评估模型的通用能力）
- 3. 数据集场景体系（数据分类体系）
- 4. 测评（获取回复方法、测评方法、指标）

## 报告结构规范

每章结构：**前置文字描述 + 表格/列表**

```
## X.X 标题
（前置文字描述，说明本节内容）

| 表格或列表内容 |
|---------------|
```

### 章节要点

1. **1.1 来源** - 前置描述 + 表格（机构、时间、链接）
2. **1.2 目标** - 前置描述 + 列表（主要目标、解决问题）
3. **1.3 应用场景** - 前置描述 + 列表
4. **1.4 数据集描述** - 前置描述 + 表格（规模、字段说明）+ 代码块示例
5. **2. 能力体系** - 表格（能力名 + 说明），指模型通用能力非数据分类
6. **3. 场景体系** - 表格（一级分类 + 包含子主题），只列分类不含数量
7. **4.1 获取模型回复** - 有模板则填写，无则用默认描述
8. **4.2 测评方法** - 说明方法类型 + 提示词模板 + 来源标注

## 注意事项

- 提示词模板优先参考论文，其次参考项目代码
- 数据统计放在 1.4 数据集描述中
- 评测指标来源需标注（论文/代码）
- 4.1 无提示词模板时填写："（无专门的提示词模板，直接将question发送给模型获取回答）"
